# 项目规则

## 项目概述
- 项目名称：公共数据运营授权运维平台
- 项目类型：智能运维平台（政务场景）
- 业务背景：政务项目运维费受7%限制，需从"人力密集型"转向"AI辅助型"
- 架构：前后端分离（轻量级应用架构）
- 前端技术栈：**React 19 + Vite**
- 后端技术栈：**FastAPI + SQLite + SQLAlchemy**
- 核心目标：降本增效、合规保障、平台稳定（99%可用性）

## 开发规范

### 代码风格
- 使用中文注释
- 变量命名使用小驼峰（camelCase）
- 组件/类命名使用大驼峰（PascalCase）

### 前端文件组织（frontend/ 目录）
- 组件放在 `frontend/src/components/` 目录
- 页面放在 `frontend/src/pages/` 目录
- 工具函数放在 `frontend/src/utils/` 目录
- 样式文件放在 `frontend/src/styles/` 目录
- API 调用放在 `frontend/src/api/` 目录

### 后端文件组织（backend/ 目录）
- 路由放在 `backend/src/routes/` 目录
- 控制器放在 `backend/src/controllers/` 目录
- 数据模型放在 `backend/src/models/` 目录
- 中间件放在 `backend/src/middleware/` 目录
- 工具函数放在 `backend/src/utils/` 目录

### AI 协作规则
1. 每次修改前先阅读相关文件
2. 修改后必须更新 CHANGELOG.md
3. 重要决策需要记录在 PRD.md 中
4. 使用 TodoWrite 工具跟踪任务进度

## 业务领域知识

### 用户角色
系统包含四类核心用户角色：

| 角色 | 职责 | 核心功能 |
|------|------|---------|
| **超级管理员** | 系统管理 | 对系统的任务功能可进行管理|
| **运维管理经理** | 全局管理、决策支持 | 驾驶舱、报表分析、全量工单管理 |
| **SRE运维工程师** | 工单执行、系统维护 | 工单处理、台账维护、自动化工具 |
| **运维需求方** | 发起运维申请 | 工单申请、进度跟踪、资源查询 |

### 角色权限说明
- 超级管理员：对系统的任务功能可进行管理
- 运维管理经理：全系统权限，可查看所有数据
- SRE运维工程师：操作权限，处理分配给自己的工单
- 运维需求方：仅查看和操作自己发起的工单

### 核心业务流程

#### 1. 运维工单生命周期流程

```mermaid
flowchart LR
    A[需求方提交申请] --> B[审批人审批]
    B --> C[SRE接单处理]
    C --> D[需求方确认关闭]
    D --> E[自动更新台账]
```

#### 2. 资源自动发放流程（预留API）

```mermaid
flowchart LR
    A[审批通过] --> B[触发自动化API]
    B --> C[预留云管平台集成接口]
    C --> D[暂不执行]
```

#### 3. 关键业务术语
- **运维工单**：运维需求的载体，包含申请、审批、执行、关闭四个阶段
- **CMDB**：配置管理数据库，记录资产、服务、账号等台账信息
- **可维性评估**：方案阶段评估系统可维护性
- **运维前置**：在交付前完成运维准备工作

## 合规与安全要求

### 政策法规依据
- **纲领性文件**：《关于加快公共数据资源开发利用的意见》
- **配套实施细则**：《公共数据资源授权运营实施规范（试行）》、《公共数据资源登记管理暂行办法》
- **上位法**：《网络安全法》、《数据安全法》、《个人信息保护法》、"数据二十条"
- **地方性规范**：《上海市公共数据资源授权运营管理办法》

### 安全要求
- 数据传输加密（HTTPS/TLS）
- 敏感数据脱敏显示
- 完整操作日志记录
- 等保2.0三级要求
- 支持水印、防截屏
- **权限控制**：RBAC权限模型，最小权限原则
- **数据备份**：定期备份，支持灾难恢复

### 审计要求
- 所有操作可追溯
- 数据变更留痕
- 定期合规审查

## 技术实现规范

### 前端开发规范
- UI组件库：Ant Design 5.x
- 状态管理：Zustand（全局）+ useState（局部）
- HTTP客户端：Fetch API / Axios
- 路由：React Router 7.x
- 表单处理：React Hook Form（推荐）
- 样式方案：CSS Modules
- **流程展示：Mermaid（项目唯一流程图语法）**

### 后端开发规范
- API设计：RESTful风格
- 认证方式：JWT Token
- 数据库操作：SQLAlchemy ORM
- 数据验证：Pydantic模型
- 异步处理：FastAPI原生async/await
- 日志记录：Python logging模块

### 接口规范
- 统一响应格式：`{code, message, data}`
- 错误码规范：HTTP状态码 + 业务错误码
- 分页规范：`page`/`page_size`/`total`
- 认证Header：`Authorization: Bearer {token}`

### 数据库设计规范
- 表名：小写下划线，如 `work_order`、`asset_config`
- 字段名：小写下划线，如 `created_at`、`updated_by`
- 主键：`id`，自增整数
- 外键：`<表名>_id`，如 `user_id`、`asset_id`
- 时间字段：`created_at`、`updated_at`
- 软删除：`is_deleted` + `deleted_at`
