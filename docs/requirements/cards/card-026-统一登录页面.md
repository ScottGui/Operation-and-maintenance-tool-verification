## 功能：统一登录页面

### 1. 这个功能是干嘛的？
提供系统统一的登录入口，所有角色（用数方、需求经理、运营方、项目经理、质量稽核经理、数据运维经理、四方组长、管理员）都从这个页面登录。

登录成功后，根据用户角色跳转到不同的工作台页面（卡片020实现角色差异化）。

### 2. 页面上有什么？

#### 登录页面整体布局
- **左侧**：系统介绍区域（可选，展示平台名称和简介）
- **右侧**：登录表单卡片

#### 登录表单卡片
- 平台名称标题："公共数据服务全生命周期管理平台"
- 副标题："运维管理系统"
- 用户名输入框（带用户图标）
- 密码输入框（带锁图标，密码显示为星号）
- 登录按钮（蓝色主按钮，带加载状态）
- 底部提示："默认账号：admin / admin123"

#### 错误提示区域
- 登录失败时显示错误信息（红字提示）

### 3. 用户怎么操作？

#### 登录流程
1. 用户打开系统，自动跳转到登录页面（未登录状态）
2. 输入用户名（如：admin）
3. 输入密码（如：admin123）
4. 点击"登录"按钮
5. 按钮显示加载状态（防止重复点击）
6. 登录成功：
   - 保存token到localStorage
   - 保存用户信息到localStorage
   - 提示"登录成功"
   - 跳转到工作台首页
7. 登录失败：
   - 按钮恢复可点击状态
   - 显示错误提示（"用户名或密码错误"）
   - 不清空输入框（方便修改）

### 4. 点完按钮发生什么？

#### 登录成功场景
- API返回：code=200，包含token和user信息
- 前端保存token和userInfo到localStorage
- 显示成功提示（message.success）
- 使用react-router的useNavigate跳转到"/"

#### 登录失败场景
| 错误情况 | 提示内容 | 处理方式 |
|---------|---------|---------|
| 用户名错误 | "用户名或密码错误" | 不明确提示是用户名还是密码错（安全考虑） |
| 密码错误 | "用户名或密码错误" | 同上 |
| 账号被禁用 | "账号已被禁用，请联系管理员" | 明确提示，因为用户需要采取行动 |
| 网络错误 | "网络异常，请稍后重试" | 可点击重试 |
| 服务器错误 | "系统繁忙，请稍后重试" | 可点击重试 |

#### 安全要求
- 密码输入框使用type="password"，显示星号
- 登录按钮在请求期间显示loading，防止重复提交
- 不记录密码到日志
- 使用HTTPS传输（后续部署时配置）

### API接口
```
POST /api/auth/login
请求体：{ username: string, password: string }
成功响应：{
  code: 200,
  message: "登录成功",
  data: {
    user: { id, username, real_name, role },
    token: "fake_token_xxx"
  }
}
失败响应：{
  code: 401,
  message: "用户名或密码错误"
}
```

### 技术要点
- 使用react-router-dom的useNavigate实现跳转
- 使用localStorage存储token和userInfo
- 使用Ant Design的Form组件做表单校验
- 使用Ant Design的message组件做提示

## 开发注意事项

1. 登录页面不需要登录就能访问（公开路由）
2. 登录成功后要根据角色信息，为后续卡片020的角色差异化做准备
3. 当前token使用假token（fake_token_{user_id}），后续卡片会接入JWT
4. 登录页面样式要美观，因为这是用户第一眼看到的页面
