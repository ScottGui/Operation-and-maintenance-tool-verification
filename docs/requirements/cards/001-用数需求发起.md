# 卡片001：用数需求发起

## 基本信息

| 项目 | 内容 |
|------|------|
| 卡片编号 | 001 |
| 卡片名称 | 用数需求发起 |
| 所属批次 | Week 2 |
| 执行角色 | 用数方 (data_consumer) |

---

## 功能概述

用数方填写数据服务需求表单，发起新的数据服务申请。这是业务流程的起点。

---

## 需求详情

### 1. 需求表单字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 需求标题 | 文本 | 是 | 简短描述需求内容，如"XX业务数据查询接口" |
| 需求类型 | 下拉选择 | 是 | 接口开发 / 数据导出 / 数据同步 / 其他 |
| 业务背景 | 多行文本 | 是 | 描述业务场景和目的 |
| 数据范围 | 多行文本 | 是 | 涉及哪些数据表/字段 |
| 使用频率 | 下拉选择 | 是 | 实时 / 每日 / 每周 / 每月 / 一次性 |
| 期望完成时间 | 日期选择 | 是 | 业务要求的上线时间 |
| 附件 | 文件上传 | 否 | 支持上传需求文档、原型图等 |

### 2. 业务流程

```
用数方填写表单 → 保存草稿 / 提交审批
                    ↓
              提交后状态：待审批
                    ↓
              流转到：需求经理
```

### 3. 页面功能

1. **新建需求按钮**：工作台快捷入口、需求管理池入口
2. **需求表单页面**：
   - 左侧：表单字段
   - 右侧：操作按钮（保存草稿、提交审批、取消）
3. **草稿功能**：可暂存未完成的表单，下次继续编辑

### 4. 权限控制

- 仅**用数方**可创建需求
- 其他角色只能查看，不能创建

## 数据库设计

### 需求单表 (work_order)

```sql
CREATE TABLE work_order (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_no VARCHAR(50) UNIQUE NOT NULL,      -- 需求单号：XQ20240325001
    title VARCHAR(255) NOT NULL,                -- 需求标题
    type VARCHAR(50) NOT NULL,                  -- 需求类型
    business_background TEXT,                   -- 业务背景
    data_scope TEXT,                            -- 数据范围
    frequency VARCHAR(50),                      -- 使用频率
    expected_date DATE,                         -- 期望完成时间
    status VARCHAR(50) DEFAULT 'pending_review', -- 状态：待审批
    
    -- 关联信息
    creator_id INTEGER NOT NULL,                -- 创建人ID
    creator_role VARCHAR(50) NOT NULL,          -- 创建人角色
    
    -- 审计字段
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- 外键
    FOREIGN KEY (creator_id) REFERENCES users(id)
);
```

### 需求单-附件关联表 (work_order_attachment)

```sql
CREATE TABLE work_order_attachment (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    work_order_id INTEGER NOT NULL,
    file_id INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (work_order_id) REFERENCES work_order(id),
    FOREIGN KEY (file_id) REFERENCES file_attachment(id)
);
```

## 接口设计

### 1. 创建需求单

```
POST /api/work-orders
```

**请求参数：**
```json
{
    "title": "XX业务数据查询接口",
    "type": "接口开发",
    "business_background": "业务场景描述...",
    "data_scope": "用户表、订单表",
    "frequency": "实时",
    "expected_date": "2026-04-30",
    "file_ids": [1, 2]  // 关联的附件ID列表
}
```

**响应：**
```json
{
    "code": 200,
    "message": "创建成功",
    "data": {
        "id": 1,
        "order_no": "XQ20260325001",
        "status": "pending_review"
    }
}
```

### 2. 获取需求单详情

```
GET /api/work-orders/:id
```

### 3. 保存草稿

```
POST /api/work-orders/draft
```
