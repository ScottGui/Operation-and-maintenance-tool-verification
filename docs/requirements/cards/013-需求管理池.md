# 卡片013：需求管理池

## 基本信息

| 项目 | 内容 |
|------|------|
| 卡片编号 | 013 |
| 卡片名称 | 需求管理池 |
| 所属批次 | Week 2 |
| 执行角色 | 所有角色（根据权限查看不同范围） |

---

## 功能概述

需求单列表管理页面，支持搜索、筛选、分页查看所有需求单。不同角色看到不同范围的需求单。

---

## 需求详情

### 1. 页面功能

| 功能 | 说明 |
|------|------|
| 列表展示 | 表格形式展示需求单基本信息 |
| 分页 | 每页10条，支持翻页 |
| 搜索 | 按需求单号、标题关键词搜索 |
| 状态筛选 | 按状态筛选需求单 |
| 类型筛选 | 按需求类型筛选 |
| 排序 | 按创建时间倒序（最新的在前） |
| 查看详情 | 点击行或按钮查看详情 |

### 2. 列表字段

| 字段 | 宽度 | 说明 |
|------|------|------|
| 需求单号 | 160px | 如 XQ20260325001 |
| 标题 | 自适应 | 需求标题，过长显示省略号 |
| 类型 | 120px | 接口开发/数据导出/数据同步/其他 |
| 状态 | 120px | 带颜色标签显示 |
| 创建人 | 120px | 创建人姓名 |
| 创建时间 | 180px | 格式：2026-03-25 10:30 |
| 操作 | 150px | 查看/编辑/删除按钮 |

### 3. 权限控制

| 角色 | 可见范围 |
|------|---------|
| 用数方 | 只看自己创建的需求单 |
| 需求经理 | 看所有待梳理和已梳理的 |
| 运营方 | 看所有（用于审批） |
| 项目经理 | 看审批通过待实施的 |
| 质量稽核经理 | 看实施完成待稽核的 |
| 数据运维经理 | 看稽核通过待试运行的 |
| 四方组长 | 看方案已提交待会审的 |
| 管理员 | 看所有 |

### 4. 状态标签颜色

| 状态 | 颜色 |
|------|------|
| 草稿 | 灰色 |
| 待审批 | 橙色 |
| 已梳理 | 蓝色 |
| 审批通过 | 绿色 |
| 已驳回 | 红色 |
| 方案已提交 | 紫色 |
| 会审中 | 橙色 |
| 实施中 | 蓝色 |
| 待质量稽核 | 橙色 |
| 待试运行 | 橙色 |
| 待监控配置 | 蓝色 |
| 已上线 | 绿色 |

## 接口设计

### 已存在接口

```
GET /api/work-orders
```

**请求参数：**
```json
{
    "page": 1,
    "page_size": 10,
    "status": "pending_review",
    "keyword": "搜索关键词"
}
```

**响应：**
```json
{
    "code": 200,
    "data": {
        "total": 9,
        "items": [...]
    }
}
```
