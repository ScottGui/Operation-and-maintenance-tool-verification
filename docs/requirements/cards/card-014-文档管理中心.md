## 功能：文档管理中心

### 1. 这个功能是干嘛的？
提供统一的文档上传、下载、管理功能。支持PDF、Word、Excel三种格式，用于存储需求全流程中的各类文档：
- 《需求规格说明书》（PDF）
- 《数据需求技术方案》（PDF）
- 《数据交维清单》（Excel）
- 《数据服务重启应急预案》（Word）
- 《数据服务自测报告》（Word）
- 《数据服务质量稽核报告》（Word）
- 《数据服务试运行报告》（Word）
- 监控配置截图（图片）

### 2. 页面上有什么？

**文档管理组件（非独立页面，嵌入在各个环节）**

#### 上传区域
- 拖拽上传区域（虚线框）
- 点击上传按钮
- 支持的文件类型提示："支持 PDF、Word、Excel 格式，单个文件不超过50MB"

#### 文件列表展示
- 文件名（带图标）
- 文件大小
- 上传时间
- 上传人
- 下载按钮
- 删除按钮（仅上传人或管理员可删除）

#### 文件图标
- PDF：红色PDF图标
- Word：蓝色Word图标
- Excel：绿色Excel图标
- 图片：紫色图片图标

### 3. 用户怎么操作？

#### 上传文档
**方式1：点击上传**
1. 点击"上传文档"按钮
2. 弹出文件选择对话框
3. 选择文件（可多选）
4. 点击确定，开始上传
5. 显示上传进度
6. 上传完成，文件显示在列表中

**方式2：拖拽上传**
1. 将文件从文件夹拖拽到上传区域
2. 文件自动上传
3. 上传完成，文件显示在列表中

#### 下载文档
1. 在文件列表中找到目标文件
2. 点击"下载"按钮
3. 浏览器开始下载文件

#### 删除文档
1. 在文件列表中找到目标文件
2. 点击"删除"按钮
3. 弹出确认对话框"确定要删除该文件吗？"
4. 点击"确定"，文件从列表中移除

### 4. 点完按钮发生什么？

#### 上传时
**成功场景：**
- 文件格式正确、大小合规 → 上传到服务器 → 显示在列表中 → 提示"上传成功"

**失败场景：**

| 错误情况 | 提示方式 | 提示内容 |
|---------|---------|---------|
| 文件格式不支持 | 上传前拦截 | "只支持 PDF、Word、Excel 格式" |
| 文件超过50MB | 上传前拦截 | "文件大小不能超过50MB" |
| 上传失败 | 上传后提示 | "上传失败，请重试" |
| 文件名重复 | 自动重命名 | "文件名_1.pdf" |

#### 删除时
- 点击删除 → 弹出确认框 → 确定 → 删除文件 → 提示"删除成功" → 列表刷新

### 技术方案
**前端：**
- 使用Ant Design的Upload组件
- 支持拖拽上传（draggable）
- 显示文件列表（fileList）

**后端：**
```
POST /api/files/upload
- 接收multipart/form-data
- 保存到uploads/{需求单ID}/目录
- 返回文件URL和元数据

GET /api/files/{fileId}/download
- 返回文件流

DELETE /api/files/{fileId}
- 删除文件（软删除）

GET /api/files?orderId={需求单ID}
- 获取某需求单的所有文档
```

**文件存储：**
- 开发阶段：本地文件系统，存放到项目目录uploads/
- 生产阶段：可迁移到对象存储（OSS/S3）

**数据库表：**
```sql
create table file_attachment (
  id INTEGER PRIMARY KEY,
  file_name VARCHAR(255),
  file_path VARCHAR(500),
  file_size INTEGER,
  file_type VARCHAR(50),
  order_id INTEGER, -- 关联的需求单ID
  uploaded_by INTEGER,
  uploaded_at DATETIME,
  is_deleted BOOLEAN DEFAULT FALSE
);
```

### 默认测试数据
无需专门测试账号，使用任意已登录账号均可上传/下载。

## 开发注意事项

1. **MVP简化**：先实现单文件上传，多文件上传可后续扩展
2. **文件命名**：上传的文件按"时间戳_原文件名"重命名，避免冲突
3. **权限控制**：用户只能删除自己上传的文件，管理员可以删除所有
4. **路径规划**：文件按需求单ID分目录存储，方便管理
