## 功能：用户与权限管理

### 1. 这个功能是干嘛的？
管理员创建和管理系统用户，给每个用户分配角色（用数方/需求经理/运营方等），控制谁能登录系统、能做什么操作。

### 2. 页面上有什么？

#### 用户列表页面
- 页面标题：用户管理
- 搜索区域：
  - 用户名/姓名搜索框（模糊搜索）
  - 角色筛选下拉框（全部/用数方/需求经理/...）
  - 状态筛选（全部/启用/禁用）
  - 搜索按钮、重置按钮
- 操作按钮：新增用户按钮
- 用户列表表格，列包括：
  - 序号
  - 用户名
  - 真实姓名
  - 所属角色
  - 账号状态（启用/禁用）
  - 创建时间
  - 操作列（编辑按钮、禁用/启用按钮、删除按钮）
- 分页组件（每页20条）

#### 新增/编辑用户弹窗
- 弹窗标题：新增用户 / 编辑用户
- 表单字段：
  - 用户名输入框（必填，唯一，英文/数字，4-20位）
  - 真实姓名输入框（必填，中文，2-20字）
  - 密码输入框（必填，新增时；可选填，编辑时。6-20位）
  - 确认密码输入框（与密码一致）
  - 角色下拉选择（必填，单选）：
    - 用数方
    - 需求经理
    - 运营方
    - 项目经理
    - 质量稽核经理
    - 数据运维经理
    - 四方组长
  - 账号状态开关（启用/禁用，默认启用）
- 按钮：保存、取消

### 3. 用户怎么操作？

#### 新增用户流程
1. 管理员点击左侧菜单"系统管理-用户管理"进入列表页
2. 点击"新增用户"按钮，弹出新增弹窗
3. 填写用户名（如：dat1）
4. 填写真实姓名（如：张三）
5. 填写密码（如：123456）
6. 确认密码（再次输入123456）
7. 选择角色（如：用数方）
8. 状态保持"启用"（默认）
9. 点击"保存"按钮

#### 编辑用户流程
1. 在用户列表中找到要修改的用户
2. 点击"编辑"按钮，弹出编辑弹窗
3. 用户名显示为灰色（不可修改）
4. 修改需要变更的字段（如：真实姓名、角色、状态）
5. 如需改密码，填写新密码；如不填，保持原密码
6. 点击"保存"按钮

#### 禁用/启用用户流程
1. 在用户列表中找到目标用户
2. 点击"禁用"（当前为启用状态）或"启用"（当前为禁用状态）按钮
3. 弹出确认框"确定要禁用/启用该用户吗？"
4. 点击确认，状态切换

#### 删除用户流程
1. 在用户列表中找到要删除的用户
2. 点击"删除"按钮（红色，带删除图标）
3. 弹出二次确认对话框：
   - 标题："确定要删除用户【xxx】吗？"
   - 提示："删除后数据将无法恢复，请谨慎操作！"
   - 按钮："确定删除"（红色危险样式）、"取消"
4. 点击"确定删除"：执行删除，提示"删除成功"，列表刷新
5. 点击"取消"：关闭对话框，无操作

**删除说明：**
- 采用软删除（标记is_deleted=true），非物理删除
- 删除后用户无法登录，但数据可恢复
- 管理员账号不可删除（保护自己）

### 4. 点完按钮发生什么？

#### 新增/编辑保存时

**成功场景：**
- 表单校验通过
- 用户名唯一性校验通过
- 关闭弹窗
- 列表自动刷新，显示新数据
- 顶部显示成功提示"用户创建成功"或"用户更新成功"

**失败场景：**

| 错误情况 | 提示方式 | 提示内容 |
|---------|---------|---------|
| 必填项为空 | 输入框红框+下方红字 | "请输入用户名" |
| 用户名已存在 | 输入框红框+下方红字 | "用户名已存在，请更换" |
| 用户名格式错误 | 输入框红框+下方红字 | "用户名只能包含字母和数字，长度4-20位" |
| 密码长度不足 | 输入框红框+下方红字 | "密码长度至少6位" |
| 两次密码不一致 | 确认密码框红框 | "两次输入的密码不一致" |
| 真实姓名格式错误 | 输入框红框 | "真实姓名只能包含中文，长度2-20字" |
| 网络错误 | 弹窗顶部提示 | "网络异常，请稍后重试" |

#### 禁用/启用操作时

**禁用流程：**
1. 点击禁用按钮
2. 弹出确认对话框："确定要禁用用户【xxx】吗？禁用后该用户将无法登录系统。"
3. 点击"确定"：状态变"禁用"，提示"禁用成功"
4. 点击"取消"：关闭对话框，无操作

**启用流程：**
1. 点击启用按钮（橙色按钮）
2. 直接启用，状态变"启用"，提示"启用成功"
（启用不需要确认，因为启用不会带来风险）

#### 删除操作时

**删除流程：**
1. 点击删除按钮（红色按钮，带删除图标）
2. 弹出确认对话框，显示警告信息
3. 点击"确定删除"：
   - 执行软删除（标记is_deleted=true）
   - 提示"删除成功"
   - 列表自动刷新，该用户消失
4. 点击"取消"：关闭对话框，无操作

**按钮颜色规范：**
| 按钮 | 颜色 | 说明 |
|------|------|------|
| 编辑 | 蓝色（link） | 普通操作 |
| 禁用 | 橙色（warning） | 警示操作，可恢复 |
| 启用 | 绿色（success） | 正向操作 |
| 删除 | 红色（danger） | 危险操作，谨慎使用 |

### 特殊说明（给AI）

1. **角色设计说明**：
   - "四方组长"是一个角色，但会有4个不同的人使用（运维、安全、平台、数据）
   - 他们在方案会审时需要各自独立审批
   - 所以测试数据里lead1-lead4都是"四方组长"角色

2. **权限简化**：
   - MVP阶段先做"角色权限"，不做"细粒度菜单权限"
   - 即：某个角色能看到哪些页面，通过前端路由控制
   - 具体每个按钮的权限控制放到P1

3. **密码存储**：
   - 后端必须加密存储（不要明文存密码）
   - 可以使用bcrypt等哈希算法

4. **初始数据**：
   - 系统启动时应该有一个默认管理员账号（admin/admin123或类似）
   - 否则第一次登录无法进行任何操作
