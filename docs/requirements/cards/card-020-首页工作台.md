## 功能：首页工作台（角色差异化）

### 1. 这个功能是干嘛的？
用户登录后看到的第一个页面，根据用户角色（7个角色）显示不同的内容、快捷入口和待办事项。

**注意**：本卡片为MVP简化版，仅实现基础的角色差异化展示。流程可视化大屏（甘特图）放到P1阶段实现。

### 2. 页面上有什么？

#### 公共部分（所有角色都有）
- **页面标题**：工作台
- **欢迎语**："欢迎回来，{真实姓名}"
- **当前角色标签**：显示用户角色

#### 统计卡片区域（3-4个卡片）
根据角色显示不同的统计数据：

| 角色 | 统计卡片1 | 统计卡片2 | 统计卡片3 | 统计卡片4 |
|------|----------|----------|----------|----------|
| 用数方 | 我的待办 | 我的需求 | 已完成的 | - |
| 需求经理 | 待梳理 | 我的梳理 | 已提交 | - |
| 运营方 | 待审批 | 待会审 | 已处理 | 今日统计 |
| 项目经理 | 待认领 | 进行中 | 已交付 | - |
| 质量稽核经理 | 待稽核 | 已稽核 | 稽核通过率 | - |
| 运维经理 | 待试运行 | 待配置监控 | 已上线 | 全局概览 |
| 四方组长 | 待会审 | 已会审 | - | - |

#### 快捷操作区域
根据角色显示不同的快捷入口按钮：

| 角色 | 快捷按钮 |
|------|---------|
| 用数方 | [发起需求] [查看我的需求] |
| 需求经理 | [查看待梳理] [查看需求池] |
| 运营方 | [待审批列表] [待会审列表] [报表中心] |
| 项目经理 | [查看待认领] [查看实施中] |

| 四方组长 | [待会审列表] [会审历史] |

#### 待办列表区域
显示当前角色需要处理的待办事项（最新5条）：

| 角色 | 待办内容 |
|------|---------|
| 用数方 | 需要确认关闭的需求、被退回的需求 |
| 需求经理 | 待认领梳理的需求 |
| 运营方 | 待审批的需求、待终审的会审 |
| 项目经理 | 待认领的需求、被退回的实施 |
| 质量稽核经理 | 待稽核的实施 |
| 运维经理 | 待试运行的实施、待配置监控的服务 |
| 四方组长 | 待会审的方案 |

### 3. 用户怎么操作？

#### 查看工作台
1. 用户登录成功
2. 系统自动跳转到工作台（/）
3. 看到与自己角色相关的统计、快捷入口、待办列表

#### 使用快捷入口
1. 点击统计卡片 → 跳转到对应列表页面（带筛选条件）
2. 点击快捷按钮 → 跳转到对应功能页面
3. 点击待办项 → 跳转到详情/处理页面

### 4. 点完按钮发生什么？

#### 统计卡片点击
- 点击"待审批 5个" → 跳转到需求池列表，自动筛选状态=待审批
- 点击"我的需求 12个" → 跳转到我的需求列表
- 其他卡片同理

#### 快捷按钮点击
- [发起需求] → 跳转到 /work-orders/create
- [待审批列表] → 跳转到 /work-orders?status=pending
- [报表中心] → 跳转到 /reports
- 其他按钮同理

#### 待办项点击
- 点击待办标题 → 跳转到详情页
- 点击"处理"按钮 → 跳转到处理页面

```
  code: 200,
  data: {
    // 根据角色返回不同的统计数据
    // 用数方示例：
    myTodo: 5,
    myOrders: 12,
    completed: 28,
    // 运营方示例：
    pendingApproval: 5,
    pendingReview: 3,
    processed: 45,
    todayStats: 8
  }
}

GET /api/dashboard/todos
响应：{

  data: [

    // 最多5条

```

### 技术要点
- 后端根据用户角色返回不同的统计数据
- 前端使用条件渲染展示不同角色的界面
- 统计数字需要实时查询（或缓存1分钟）
- 待办列表按优先级排序
