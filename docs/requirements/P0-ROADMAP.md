### P1-001 流程可视化大屏详细需求

**提出时间**：2026-03-25（开发过程中提出）  
**提出人**：运维经理（scott）  
**需求背景**：希望直观看到当前所有工单在各个节点的分布情况

**具体设计**：
```
Dashboard顶部流程可视化区域：
┌─────────────────────────────────────────────────────────────────────────┐
│  流程监控大屏                                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  [待审批] ──▶ [已梳理] ──▶ [审批通过] ──▶ [方案已提交] ──▶ [会审中]     │
│   23个      15个          8个            5个              3个           │
│   35%       23%           12%            8%               5%            │
│                                                                         │
│  [实施中] ──▶ [待质量稽核] ──▶ [待试运行] ──▶ [待监控配置] ──▶ [已上线]  │
│   2个        1个            0个             0个               156个      │
│   3%         2%             0%              0%                237%       │
│                                                                         │
│  总工单数：223  |  进行中：67  |  已完成：156                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**功能要求**：
1. 横向甘特图形式展示10个流程节点
2. 节点之间用箭头连接，显示流转方向
3. 每个节点显示：
   - 节点名称（如"待审批"）
   - 当前工单数量（如"23个"）
   - 完成百分比（如"35%"）
4. 点击节点可跳转到该状态的工单列表
5. 底部显示总体统计：总工单数、进行中、已完成

**技术方案**：
- 前端：使用ECharts或自定义CSS实现甘特图
- 后端：新增API `/api/statistics/flow-status` 返回各状态工单数
- 实时性：页面加载时查询，可手动刷新

**依赖**：需等待P0阶段全部流程节点实现完成后才能开发
