## 📝 卡片命名规范

```
card-XXX-功能名称.md

示例：
├── card-015-用户与权限管理.md
├── card-026-统一登录页面.md
├── card-001-用数需求发起.md
```

**编号规则：**
- 编号 = 卡片编号，3位数字，不足补零
- 按开发顺序排列，不是按业务顺序

## 🔄 版本管理规则

### 什么时候创建新版本？

| 场景 | 操作 |
|------|------|
| 卡片首次创建 | 直接放入 `cards/` 目录 |
| 卡片需求变更（未开发） | 直接修改 `cards/` 中的文件，记录变更日志 |
| 卡片需求变更（已开发完成） | 1. 原文件移入 `archive/vX.Y/`  <br>2. 创建新文件放入 `cards/`  <br>3. 更新 `versions.md` |
| 整体需求大版本更新 | 所有卡片移入 `archive/vX.Y/`，重新创建新批次 |

### 版本号规则

```
v主版本.次版本

示例：
├── v1.0    # 初始版本（MVP阶段第一版）
├── v1.1    # 小修订（某几张卡片调整）
├── v2.0    # 大改版（P1阶段，新增功能）
```

### 卡片文件内的版本标记

每张卡片顶部必须包含：

```markdown
---
卡片编号：015
卡片名称：用户与权限管理
当前版本：v1.0
创建日期：2026-03-24
最后更新：2026-03-24
状态：待开发  # 待开发 / 开发中 / 已完成
---
```

## 🔄 如何变更需求卡片

### 场景1：卡片还没开发，想修改

1. 直接修改 `cards/card-XXX-功能名称.md`
2. 更新文件头部的 `最后更新` 日期
3. 在 `变更记录` 表格中添加一行
4. 如果改动较大，可以考虑版本号+0.1（v1.0→v1.1）

### 场景2：卡片已开发完成，想修改

1. 在 `archive/` 下创建版本文件夹（如 `v1.0/`）
2. 将原文件复制到 `archive/v1.0/`
3. 在 `cards/` 中创建新文件，版本号+1（v1.0→v2.0）
4. 在 `versions.md` 中记录变更原因
5. 通知AI基于新卡片重新开发（或修改已有代码）
